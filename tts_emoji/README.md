# tts_emoji

発声効果の絵文字（Irodori の Emoji Palette）を **どう付与するか** を決めるグローバル設定を置く場所です。
キャラクターごとの設定（`Character/*/profile.json` や Options 画面）とは別で、**全キャラ共通**です
（絵文字が Irodori 上でどう鳴るかは、どのキャラが喋っても同じ性質のため）。

## 背景

返答は「感情セグメント → 文チャンク」に分割し、チャンクごとに独立して TTS 生成します。
発声効果の絵文字はテキスト先頭に埋め込む「効果マーク」で、**全チャンクに前置すると文ごとに
効果音が鳴り**、「文と文の間の意味不明な発声」に聞こえます。そこで絵文字を 2 種類に分けます。

- **singleShot（単発音）**: 吐息・喘ぎ・泣き声・咳・あくびなど、その場で 1 回鳴る非言語音。
  → **セグメント先頭の 1 文だけ** に付与（繰り返しを防ぐ）。
- **sustained（持続系）**: 話し方・声色・音響効果（ゆっくり／早口／朗読／優しく／電話越し／
  エコー／口を塞ぐ／各種の感情表現など）。
  → **全チャンク** に付与（セグメント途中で話し方がぶれないように）。

※ 感情そのもの（caption）は分類に関わらず全チャンクに効き続けます。

## ファイル

- `emoji_behavior.json` … 分類の上書き設定。
  - `singleShot` / `sustained` … それぞれの絵文字リスト。
  - `defaultForUnknown` … どちらにも載っていない絵文字（**新しく追加された絵文字を含む**）の既定。
    安全側の `singleShot` 推奨（効果音が全文で繰り返す不具合の再発を防ぐ）。

### 仕様

- コードに組み込みの既定分類（パレットのラベル基準）を土台に、このファイルの内容を
  **絵文字単位でマージ（上書き）** します。書いていない絵文字はコード既定のまま。
- 同じ絵文字を両リストに書いた場合は **singleShot を優先**。
- パレットに存在しない絵文字は無視し、ログに出します。
- 新しい絵文字が Irodori 側に追加されると、起動ログに
  `[emoji-behavior] unclassified (default=singleShot): 🆕(ラベル)` のように出るので、
  必要に応じてこのファイルへ分類を追記してください。

### 環境変数

- `TTS_EMOJI_BEHAVIOR_FILE` … 参照するファイル名（既定 `emoji_behavior.json`）。
  相対名は `tts_emoji/` → プロジェクト直下 → カレントの順で解決します。
